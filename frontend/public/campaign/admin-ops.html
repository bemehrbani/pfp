1. Django Admin Login

The Django Admin interface is the primary backend tool for managing campaigns, tasks, volunteers, and proofs.

Step 1

Open the Admin Panel

Navigate to:

https://peopleforpeace.live/admin/
Step 2

Log In

Use the admin credentials provided by the project lead. After login, you'll see the admin dashboard with sections for:

  • Campaigns — Campaign settings, volunteer lists
  • Tasks — Task definitions, key tweets, task assignments
  • Telegram — Telegram users and settings
  • Analytics — User actions, activity tracking
Tip: Bookmark the admin URL. If you can't log in, ask the project lead to create an account via python manage.py createsuperuser.

2. Managing Tweet Comment Targets

The bot shows volunteers specific tweets to comment on. These are stored as Key Tweets linked to the "Comment on Key Tweets" task.

⚠️ Why this matters: If the target accounts are inactive (no recent tweets) or the tweet URLs are broken, volunteers see dead links and can't complete their task.

Finding Active Tweets to Target

Search Twitter/X for recent tweets about Minab or Iran children using these queries:

#JusticeForMinabChildren #168Children Minab school airstrike Iran children war crime

Look for tweets from journalists, NGOs, or accounts with 1K+ followers that have recent activity (within the last 24–48 hours).

Updating Key Tweets in Admin

Step 1 — Open Key Tweets

Navigate to Tasks → Key Tweets

In Django Admin, click Tasks in the sidebar, then Key tweets.

Step 2 — Deactivate Old Tweets

Uncheck "Is Active" on stale entries

  1. Find entries where the tweet is old or the account is inactive
  2. Click on the entry
  3. Uncheck Is Active
  4. Click Save

Alternatively, select multiple entries → Action: "Mark as inactive"

3. Reviewing Volunteer Proofs

When volunteers complete tasks, they submit proof (tweet URL, screenshot, or text). These need to be reviewed.

Step 1

Open Task Assignments

In Django Admin → Tasks → Task assignments. Filter by Status = pending to see unreviewed proofs.

Step 2

Review Each Proof

  1. Click on a pending assignment
  2. Check the proof_text field (contains the tweet URL or message)
  3. If they submitted a screenshot, check proof_image
  4. Open the tweet URL in a browser to verify it exists and matches the task
Step 3

Approve or Reject

Change the Status field:

  • verified — Proof is valid, task is complete
  • rejected — Proof doesn't meet requirements

Click Save.

Review Criteria:
  • ✅ Tweet URL works and links to a real post
  • ✅ Content is relevant to the campaign (mentions Minab, children, peace)
  • ✅ Not a retweet of their own testing tweet
  • ❌ Reject if URL is broken, content is unrelated, or clearly fake

4. Managing Tasks

Editing a Task

Django Admin → Tasks → Tasks → Click on the task to edit.

Key fields:

FieldWhat It Does
title / title_fa / title_arTask name in each language
descriptionDetailed instructions (EN/FA/AR)
task_typeType: twitter_post, twitter_comment, content_creation, etc.
target_urlDefault URL for the task (e.g., petition link)
is_activeToggle to show/hide task from volunteers
orderDisplay order in the task list (lower = higher)
pointsPoints awarded on completion
estimated_timeTime estimate shown to volunteers
Tip: To temporarily hide a task without deleting it, uncheck Is Active. The task and all its assignments will remain in the database.

5. Admin Bot Commands

These commands only work for users whose Telegram ID is in the BOT_ADMIN_IDS environment variable.

CommandWhat It Does
/post_dashboardPost the campaign dashboard to the channel and pin it
/refresh_dashboardForce-update the pinned dashboard message
/send_reminder 1dSend a storm reminder (1w / 1d / 1h)
/stormStart the Twitter Storm tweet-serving flow
/leaderboardShow the top volunteers leaderboard

Testing Bot Commands

To test admin commands, send them directly to the bot in a private chat. They won't be visible to other users.

⚠️ Admin Access: To add a new admin, their Telegram user ID must be added to BOT_ADMIN_IDS in the server's .env file, then the bot container must be restarted.

6. Docker & Logs

The platform runs as Docker containers on the server. Here are common maintenance commands.

Checking Service Status

ssh user@65.109.198.200 cd /opt/pfp docker compose ps

Viewing Logs

# Bot logs (last 100 lines, follow) docker compose logs -f --tail=100 telegram-bot # Backend API logs docker compose logs -f --tail=100 backend # All services docker compose logs -f --tail=50

Restarting Services

# Restart just the bot docker compose restart telegram-bot # Restart everything docker compose down && docker compose up -d # Full rebuild (after code changes) docker compose up --build -d
ProblemWhat to CheckFix
Bot not respondingdocker compose logs telegram-botdocker compose restart telegram-bot
Dashboard not updatingdocker compose logs celery-beatdocker compose restart celery-beat
Tasks not loadingdocker compose logs backenddocker compose restart backend
Database errorCheck migrationsdocker compose exec backend python manage.py migrate
⚠️ Never run docker compose down -v — the -v flag deletes database volumes. Always use docker compose down (without -v) followed by docker compose up -d.
