1. Platform Overview

The People for Peace (PFP) platform enables volunteers worldwide to take micro-actions for the Justice for Minab Children campaign. The platform consists of:

  • Telegram Bot (@peopleforpeacebot) — primary volunteer interface
  • Telegram Channel (@people4peace) — public campaign updates & dashboard
  • Website (peopleforpeace.live) — memorial, evidence, gallery, legal docs
  • Django Admin — backend for managing campaigns, tasks, volunteers, and proofs

2. Architecture & Tech Stack

Backend

  • Python / Django — core framework
  • python-telegram-bot — Telegram Bot API integration (polling mode in dev, webhook in prod)
  • PostgreSQL — primary database
  • Celery + Redis — background tasks (dashboard refresh every 60 min)

Frontend (Static)

  • Vanilla HTML/CSS/JS — all pages are static files served from Nginx
  • Dark theme with Playfair Display + DM Sans fonts
  • No JavaScript framework — keeps pages fast and simple

Infrastructure

  • VPS (SkinScope Server) — Ubuntu, hosts both Django and static files
  • Nginx — reverse proxy + static file serving
  • Supervisor — process management for Django, Celery worker, Celery Beat
  • Let's Encrypt — SSL for peopleforpeace.live

5. Translation System

All user-facing text lives in telegram-bot/utils/translations.py as a Python dictionary (TRANSLATIONS). Three supported languages:

  • EN — English (default)
  • FA — Farsi (فارسی) — right-to-left
  • AR — Arabic (العربية) — partial, falls back to EN

Adding a New Translation Key

# In translations.py → TRANSLATIONS dict "new_key": { "en": "English text here", "fa": "متن فارسی اینجا", "ar": "نص عربي هنا" },
RTL Note: Telegram handles RTL rendering automatically based on the script used. No special markup needed for Farsi/Arabic messages.

6. Admin & Moderation

Django Admin is available at /admin/ on the backend. Key models:

  • Campaign — title, description, status, channel ID, dashboard message ID
  • Task — belongs to campaign, type, instructions, time estimate, points
  • Volunteer / TelegramUser — auto-created on /start, tracks joined campaigns
  • TaskCompletion — proof text/media, status (pending/approved/rejected)

Proof Review Workflow

  1. Volunteer submits proof → status = pending
  2. Admin reviews in Django Admin → approve or reject
  3. Points awarded on approval (future: auto-award on approve)

7. Deployment Pipeline

Backend (Bot + API)

# SSH into server ssh your-user@server # Pull latest code cd /path/to/pfp && git pull # Restart services sudo supervisorctl restart pfp-bot pfp-celery pfp-beat # Check status sudo supervisorctl status

Frontend (Static Files)

# Frontend files are served directly by Nginx # After editing, just push to the server: scp -r frontend/public/* user@server:/var/www/peopleforpeace.live/ # Or if using git on the server: cd /var/www/peopleforpeace.live && git pull
⚠️ Important: Always test locally before deploying. Use the Telegram test bot for development. Never deploy untested code to the production bot.
